**ASSISTANT:** Yes — and clarified that we are not currently using Poetry at all. We are already on `pyproject.toml` + hatchling, which is the correct foundation. The question is whether to adopt uv as the development workflow tool on top of that.

uv (by Astral) replaces pip, pip-tools, venv, and virtualenv in a single fast binary. It is 10–100x faster than pip, generates a `uv.lock` for reproducible installs, and is rapidly becoming the standard in the Python ecosystem.

Key commands:
```bash
uv sync --group dev    # create .venv + install deps
uv run pytest tests/   # run tests
uv add --dev <pkg>     # add dev dependency
uv lock                # regenerate lock file
```
